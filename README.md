# Karmazin Tattoo — сайт + Decap CMS (git-based админка)

Это уже не localStorage-версия. Теперь правки из админки реально сохраняются
в файл `content/site.json` в вашем git-репозитории, и видят их ВСЕ посетители
сайта — не только вы в своём браузере.

Дальше сайт не нужно никогда больше вручную перезаливать — просто редактируете
через `/admin`, а Netlify сам пересобирает и обновляет живой сайт.

## Разовая настройка (займёт ~15 минут)

### 1. Создайте аккаунт на GitHub (если ещё нет)
https://github.com/signup — бесплатно.

### 2. Создайте новый репозиторий
- New repository → назовите например `karmazin-tattoo-site`
- Public или Private — не важно, оба варианта бесплатны
- Загрузите туда ВСЕ файлы из этой папки (можно через "Add file → Upload files"
  в вебе GitHub, просто перетащив всю папку — включая скрытые вложенные папки
  `admin/` и `content/`)

### 3. Подключите репозиторий к Netlify
- https://app.netlify.com → Add new site → Import an existing project
- Выберите GitHub → авторизуйтесь → выберите ваш репозиторий `karmazin-tattoo-site`
- Build command: оставьте пустым
- Publish directory: `/` (корень)
- Deploy site

Через минуту сайт будет доступен на случайном адресе вроде
`https://random-name-123.netlify.app`.

### 4. Включите Identity (это система входа в админку)
В панели Netlify для вашего сайта:
- Site configuration → Identity → Enable Identity
- Identity → Registration preferences → выберите **Invite only**
  (чтобы левые люди не могли сами себе зарегистрироваться как админ)

### 5. Включите Git Gateway (это то, что даёт админке право сохранять в git)
- Там же: Identity → Services → **Enable Git Gateway**

### 6. Пригласите себя как пользователя
- Identity → Invite users → впишите свой email
- В почте придёт письмо-приглашение → перейдёте по ссылке → зададите пароль

### 7. Зайдите в саму админку
- Откройте `https://ваш-сайт.netlify.app/admin/`
- Войдёте под своим email/паролем
- Увидите форму со всеми полями сайта — редактируете, жмёте **Publish** →
  через 30–60 секунд изменения увидят все посетители сайта

## Что дальше можно менять только через /admin, а не вручную:
- все тексты (герой, о мастере, стили, шаги записи)
- цены (грн)
- фото в галерее (можно сразу загружать новые файлы прямо из формы)
- цвета/градиент сайта
- телефон и ссылку на Instagram

## Технические детали (на будущее, если будете сами админить репозиторий)
- `content/site.json` — единственный файл с данными, который реально меняется
- `content.default.js` — аварийный запасной контент на случай, если `site.json`
  не подгрузится (например, вы открыли `index.html` локально с диска без сервера)
- `admin/config.yml` — описание полей формы для Decap CMS; если захотите добавить
  новое поле на сайт — сначала добавляете его в `index.html`/`content.default.js`,
  потом прописываете в `config.yml`, чтобы оно появилось в форме админки
